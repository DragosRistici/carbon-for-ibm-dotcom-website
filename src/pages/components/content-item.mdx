## Overview

The content item component can be customized for different types of web experiences. It can be used multiple times
inside of a content group as a list of items of the same type.

![A visual example of the content item component](../../images/component/content-item/component--content-item--overview.png)

<Caption>Anatomy of the content item component</Caption>

<br />

1. **Heading:** Customizable heading for the content item component.
2. **Media (image or video):** You can add image or video inside of a content item.
3. **Description:** Use this option for a short description of the content item.
4. **CTA text link, or button:** Use this as a call-to-action at content item level.

#### When to use

Use content item when you need to organise a group of content elements into a content unit. It can live inside of a
content group component. One of the advantages of the content item is that it is consistent throughout the IBM dotcom
pages.

#### When not to use

Do not look at content item as a replacement for a content block or a content group. Do not add it inside a content
section’s children container. A content item should not live directly inside a content section due to hierarchy
conflicts.

## Behaviors

The content item elements are persistent throughout the online experience. It is fully responsive and changes how
elements are displayed based on the browser size.

#### Desktop breakpoints (max, xlg, and lg)

If the browser is wider than the max breakpoint of 1584px, the content item and all other page content will center and
extra margin will appear on either side of the page layout. The content item usually takes the centered 8 columns of the
grid. It is possible to layout a content item on 4 columns, making the reading experience in certain situations more
efficient. If you need item groupings, it is recommended to use the content group component.

![A visual example of the content item component at desktop breakpoints](../../images/component/content-item/component--content-item--behaviors-desktop.png)

<Caption>Content item examples, displayed on 8 grid columns (above) or 4 grid columns (below)</Caption>

#### Md and sm breakpoints

For the medium breakpoint, the content item may take all 8 or, as seen above, just 4 grid columns. For the small
breakpoint, the content item will always take all 4 grid columns.

![A visual example of the content item component at mobile breakpoints](../../images/component/content-item/component--content-item--behaviors-mobile.png)

<Caption>Content items at md 672px and sm 320px breakpoints</Caption>

## Tips and techniques

The content item elements are persistent throughout the online experience. It is fully responsive and changes how
elements are displayed based on the browser size.

- Important: always keep multiple related content items in a content group component.
- If you need to add different types of content items, use a content group component for each type. It’s ok (but not
  recommended) to have a content group with only one content item.
- It is recommended to maintain a hierarchical approach when adding content to a web page. Start with a content section,
  add content blocks to it, then add content groups inside the content blocks and, in the end, add your content items
  inside the content groups. Be consistent in this approach.
